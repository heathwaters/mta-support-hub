VA Support Agent, Admin Access Requirements

Developer handoff document · What to build so our VA can handle customer support

1 Background

We are hiring a VA (Virtual Assistant) to handle live customer support chats via tawk.to for both Match Tennis App (MTA) and Match Tennis Team (MTT). The VA needs limited admin access to resolve common customer issues without full database access.

The VA already has:

  • A searchable Knowledge Base (HTML file) with 126 scripted customer responses
  • Ticket creation workflow for issues beyond their access level
  • tawk.to chat access to communicate with customers

What the VA still needs:

  • A simple admin interface (web page or panel) where they can look up customer accounts and perform a few specific actions

2 What the VA Needs to Do

The VA needs access to perform only these specific actions. Nothing more.

4 Suggested Screen Layout

A single page web app with these sections would cover everything:

Screen 1: Account Lookup

  • Search bar at top: search by email, name, or USTA number
  • Account info card showing: name, email, account status (active/locked), date created, privacy setting, age division
  • ⚠️ NO financial data shown: No subscription plan, no billing dates, no payment info. VA creates tickets for all billing questions.
  • Action buttons:
    • Send Password Reset — triggers reset email, shows copyable link
    • Unlock Account — only visible if account is locked
    • Set to Private — with confirmation dialog ("Are you sure? 6 month lock")

Screen 3: Tournament Lookup

  • Search bar: Search by Tournament ID
  • Tournament card: Tournament name, dates, location, director name
  • Action buttons:
    • Copy Waiver Link — copies the pre-tournament check-in / VTD waiver link for this tournament. VA pastes into tawk.to chat.
    • Copy Tournament Link — copies the USTA tournament page link
  • Read only tournament info, no editing

Screen 4: Player Profiles (below tournament or as a tab)

  • List of players linked to the account
  • Each player shows: name, USTA number, NTRP rating, UTR rating, privacy status
  • Read only, no edit buttons

Screen 5: Recent Activity (optional, very helpful)

  • Recent tournaments registered
  • Recent match results
  • Read only, helps VA verify customer complaints like "my match is missing"

5 Match Tennis Team (MTT), Additional Actions

The admin panel also needs a Match Tennis Team section (or separate tab) for MTT-specific actions. These are separate from the MTA actions above.

ActionDetailsPriority
Send team registration link Look up a team event by state/date, then generate and copy the registration link to paste into tawk.to chat. VA frequently needs to send customers the correct registration link for their specific event. Must have
Send password reset link (MTT) Same as MTA but for Match Tennis Team accounts. Trigger a reset email and provide a copyable link. MTT has separate login credentials from MTA. Must have
Update NTRP rating Edit a player's NTRP (National Tennis Rating Program) rating on their MTT profile. Needed when a player's rating has changed or was entered incorrectly during registration. Must have

Suggested MTT Screen Layout

  • Search bar: Search MTT accounts by email, name, or team name
  • Account card: Name, email, teams they belong to, role (player/captain)
  • Action buttons:
    • Send Password Reset — triggers MTT-specific reset email with copyable link
    • Update NTRP — dropdown or input to change rating
    • Update T-shirt Size — dropdown: YS, YM, YL, AS, AM, AL, AXL, AXXL
  • Event registration links:
    • Search/filter events by state and date
    • Copy Registration Link button next to each event
    • VA pastes link into tawk.to for the customer
  • Player/team info: Read only view of team roster, division, event details

6 Security Requirements

  • Login required: VA must log in with their own credentials (not a shared account)
  • Role based: Create a "Support Agent" role with only the permissions listed above
  • Audit log: Log every action the VA takes (who searched what, who sent resets, who changed privacy) with timestamps
  • No bulk actions: VA can only act on one account at a time
  • Session timeout: Auto logout after 30 minutes of inactivity
  • Access from: Web browser only (no API keys or direct database access given to VA)

7 How It Gets Used Day to Day

Here's a typical workflow showing how the VA uses tawk.to + the Knowledge Base + the admin panel together:

Example: Customer says "I'm locked out of my account"

StepWhereWhat happens
1tawk.toCustomer sends message. VA greets them.
2Support HubVA searches "locked out" in the KB. Finds the entry, copies the customer response.
3tawk.toVA pastes the response asking for name and email.
4tawk.toCustomer replies with their email.
5Admin PanelVA searches by email. Sees account is locked. Clicks "Unlock Account."
6Admin PanelVA clicks "Send Password Reset." Copies the reset link.
7tawk.toVA pastes: "I've unlocked your account and sent a password reset link. Here's the direct link: [link]"
8tawk.toCustomer confirms it worked. VA closes chat.

Example: Customer says "I was charged twice, I want a refund"

StepWhereWhat happens
1tawk.toCustomer sends message. VA greets them.
2Support HubVA searches "charged twice." Finds the entry. Sees it's "DB Required."
3tawk.toVA copies and pastes the customer response asking for account details.
4Support HubAdmin Notes say: "THIS IS A FINANCIAL/BILLING ISSUE. CREATE A SUPPORT TICKET IMMEDIATELY."
5Support HubVA goes to Create Ticket tab. Fills in customer info and what the customer described.
6VA does NOT look up any billing info in admin panel. Zero financial access.
7tawk.toVA tells customer: "I've created ticket #1234 for our billing team. They will review your account and follow up within 1 business day."
