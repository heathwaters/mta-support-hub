VA Admin Panel, API Specification

Developer blueprint · Build these API endpoints so the VA admin panel can connect to your database

Overview

What this document is

This is the API specification for the VA Support Admin Panel. You (the developer) build these endpoints. We have already built the frontend that will call them.

Architecture

┌─────────────────┐     HTTPS/JSON      ┌─────────────────┐      SQL       ┌──────────────┐
│  VA Admin Panel  │ ──────────────────► │   Your API       │ ────────────► │  Your DB      │
│  (React frontend)│ ◄────────────────── │   (you build)    │ ◄──────────── │  (MySQL/etc)  │
└─────────────────┘                      └─────────────────┘               └──────────────┘
     Hosted on                            Hosted on your                    Your existing
     Netlify/Vercel                       server/cloud                      database

Requirements

  • Base URL: Your API will live at something like https://api.matchtennisapp.com/va or https://your-server.com/api/va
  • Auth: Every request must include an API key or JWT token in the header: Authorization: Bearer {token}
  • Format: All requests and responses are JSON
  • CORS: Allow requests from the admin panel domain
  • Logging: Log every API call with: timestamp, VA user ID, action, target account email, IP address

Endpoint 0: Authentication

POST /va/auth/login

VA logs into the admin panel with their own credentials.

Request

{
  "email": "va@matchtennisapp.com",
  "password": "their-password"
}

Response

{
  "success": true,
  "token": "eyJhbGciOiJIUzI1NiIs...",   // JWT token, expires in 8 hours
  "agent": {
    "name": "VA Agent Name",
    "role": "support_agent"
  }
}
Security: Create a "support_agent" role in your system. This role can ONLY access the endpoints listed in this document. No access to billing, financial data, admin settings, or destructive actions.

Endpoint 1: Search Accounts (MTA)

GET /va/mta/accounts/search?q={query} Must have

Search MTA accounts by email, name, or USTA number.

Request

GET /va/mta/accounts/search?q=john@example.com
Authorization: Bearer {token}

Response

{
  "results": [
    {
      "id": "usr_12345",
      "name": "John Smith",
      "email": "john@example.com",
      "account_status": "active",         // active, locked, suspended
      "failed_login_attempts": 0,
      "privacy": "public",                // public, private
      "date_created": "2024-01-15",
      "players": [
        {
          "id": "plr_789",
          "name": "Jane Smith",
          "usta_number": "12345678",
          "age_division": "14s",
          "utr_rating": "5.23",
          "ntrp_rating": "4.0",
          "utr_last_synced": "2025-03-28"
        }
      ]
    }
  ],
  "total": 1
}
Important: Do NOT return any billing, subscription, payment, or financial data. The VA has zero access to financial information.

Endpoint 2: Send Password Reset (MTA)

POST /va/mta/accounts/{id}/password-reset Must have

Trigger a password reset email and return a copyable reset link.

Request

POST /va/mta/accounts/usr_12345/password-reset
Authorization: Bearer {token}

Response

{
  "success": true,
  "reset_link": "https://www.matchtennisapp.com/reset?token=abc123def456",
  "email_sent_to": "john@example.com",
  "expires_in": "24 hours"
}
The VA copies reset_link and pastes it into the tawk.to chat so the customer can click it directly, especially useful when reset emails aren't arriving.

Endpoint 3: Unlock Account / Clear Failed Attempts (MTA)

POST /va/mta/accounts/{id}/unlock Must have

Unlock a locked account and clear all failed login attempts.

Request

POST /va/mta/accounts/usr_12345/unlock
Authorization: Bearer {token}

Response

{
  "success": true,
  "previous_status": "locked",
  "new_status": "active",
  "failed_attempts_cleared": 5
}

Endpoint 4: Set Account to Private (MTA)

PUT /va/mta/accounts/{id}/privacy Important

Change account privacy from public to private. One-way action with 6 month lock.

Request

PUT /va/mta/accounts/usr_12345/privacy
Authorization: Bearer {token}
{
  "privacy": "private",
  "confirmed": true        // frontend must confirm with VA before sending
}

Response

{
  "success": true,
  "privacy": "private",
  "locked_until": "2026-10-04",     // 6 months from now
  "warning": "Cannot be reversed until 2026-10-04"
}
Backend must enforce: If the account is already private and the lock period hasn't expired, reject the request to change back to public.

Endpoint 5: Update UTR Rating (MTA)

POST /va/mta/players/{id}/utr-sync Important

Trigger a UTR rating resync for a specific player.

Request

POST /va/mta/players/plr_789/utr-sync
Authorization: Bearer {token}

Response

{
  "success": true,
  "player": "Jane Smith",
  "previous_utr": "5.23",
  "sync_status": "queued",       // queued, completed, failed
  "message": "UTR sync triggered. Rating will update within 24-48 hours."
}

Endpoint 6: Add/Update Age Division (MTA)

PUT /va/mta/players/{id}/age-division Important

Set or update a player's primary age division.

Request

PUT /va/mta/players/plr_789/age-division
Authorization: Bearer {token}
{
  "age_division": "14s"       // 10s, 12s, 14s, 16s, 18s
}

Response

{
  "success": true,
  "player": "Jane Smith",
  "age_division": "14s"
}

Endpoint 7: Search Tournament + Get Waiver Link (MTA)

GET /va/mta/tournaments/search?tid={tournament_id} Must have

Look up a tournament by ID and return the waiver/VTD check-in link.

Request

GET /va/mta/tournaments/search?tid=TN12345
Authorization: Bearer {token}

Response

{
  "tournament": {
    "id": "TN12345",
    "name": "USTA Southern Junior Open",
    "dates": "April 12-14, 2026",
    "location": "Orlando, FL",
    "director_name": "Chris Walling",
    "director_email": "director@example.com",
    "waiver_link": "https://www.matchtennisapp.com/baseapp/checkin?tid=TN12345",
    "usta_link": "https://playtennis.usta.com/competitions/TN12345"
  }
}
The VA copies waiver_link and pastes into tawk.to. This is the pre-tournament check-in / VTD registration link that all players must complete.

Match Tennis Team (MTT) Endpoints

These are separate endpoints for the MTT system, which has its own accounts and database.

Endpoint 8: Search Accounts (MTT)

GET /va/mtt/accounts/search?q={query} Must have

Search MTT accounts by email, name, or team name.

Response

{
  "results": [
    {
      "id": "mtt_usr_456",
      "name": "John Smith",
      "email": "john@example.com",
      "username": "john@example.com",
      "account_status": "active",
      "failed_login_attempts": 0,
      "teams": [
        {
          "team_name": "Orlando Tennis Club A",
          "role": "player",              // player, captain, co-captain
          "event_name": "USTA Southern Adult 18+",
          "ntrp_rating": "4.0",
          "tshirt_size": "AL"
        }
      ]
    }
  ]
}

Endpoint 9: Send Password Reset (MTT)

POST /va/mtt/accounts/{id}/password-reset Must have

Same pattern as MTA Endpoint 2, but for MTT credentials. Return a copyable reset link.

Response

{
  "success": true,
  "reset_link": "https://www.matchtennisteam.com/baseapp/resetpw?token=xyz789",
  "email_sent_to": "john@example.com",
  "expires_in": "24 hours"
}

Endpoint 10: Unlock Account / Clear Failed Attempts (MTT)

POST /va/mtt/accounts/{id}/unlock Must have

Same pattern as MTA Endpoint 3.

Endpoint 11: Get Team Registration Link (MTT)

GET /va/mtt/events/search?state={state}&date={date} Must have

Search for team events by state and/or date, return registration links.

Request

GET /va/mtt/events/search?state=FL&date=2026-04
Authorization: Bearer {token}

Response

{
  "events": [
    {
      "id": "tn_706",
      "name": "MS JTT Spring 2026",
      "state": "FL",
      "dates": "April 19-20, 2026",
      "registration_link": "https://mstennis.matchtennisteam.com/baseapp/online/tourninfo.html?tid=tn_706",
      "registration_guide": "https://matchtennisteam-site-assets.s3.amazonaws.com/MatchTeamAppRegistrationGuideGenral.pdf",
      "director_name": "Tournament Director",
      "director_email": "director@example.com"
    }
  ]
}
VA copies registration_link and pastes into tawk.to. Can also share the registration_guide PDF link if the customer needs step-by-step instructions.

Update a player's NTRP rating for a specific team.

Request

Authorization: Bearer {token}

Response

  "player": "John Smith",
  "team": "Orlando Tennis Club A",
  "previous_ntrp": "4.0",
  "new_ntrp": "4.5"
}

What to Give Back to Us

Once the API is built, we need:

  1. Base URL — where the API lives (e.g., https://api.matchtennisapp.com/va)
  2. VA login credentials — username/password for the support agent account
  3. Any changes to the response format from what's documented above
  4. CORS whitelist — we'll tell you the admin panel domain to allow

With these 4 things, we can connect the frontend admin panel to your API and the VA is ready to go.
